# Contributing to Toward Standardization

Thank you for your interest in contributing. This project is intended to grow through community discussion, dataset suggestions, documentation, software improvements, and new benchmark extensions.

## Ways to contribute

You can contribute by:

- opening an issue with a dataset suggestion
- commenting on benchmark methodology
- proposing a reference standard for a model system
- improving documentation
- testing the software on your system
- improving plots and reports
- submitting code through pull requests
- suggesting extensions to other tissues or model systems

## Before contributing code

Please open an issue first if your change is large or changes the methodology. This helps align the contribution with the project goals before work begins.

## Dataset suggestions

Dataset suggestions should include:

- accession or source link
- tissue or model type
- species
- whether samples are healthy, mock, untreated, baseline, perturbed, or disease-associated
- RNA-seq format availability
- whether raw counts or count-like data are available
- why the dataset is relevant to a benchmark

## Pull requests

Pull requests should:

- describe the purpose of the change
- explain whether results are expected to change
- include documentation updates when needed
- avoid committing large temporary output files unless they are intentionally part of an example

## Conduct

Contributors are expected to participate respectfully and constructively. See `CODE_OF_CONDUCT.md`.
